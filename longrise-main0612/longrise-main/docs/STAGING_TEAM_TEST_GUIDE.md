## 1. 접속 주소

아래 주소로 바로 접속하면 됩니다.

| 구분 | 주소 | 확인 상태 |
|---|---|---|
| 사용자 프론트 | http://43.201.76.192:5173 | 접속 확인 완료 |
| 관리자 프론트 | http://43.201.76.192:5174 | 접속 확인 완료 |
| API 상태 확인 | http://43.201.76.192:8000/health | `healthy`, `staging` 확인 완료 |

주의: 현재 주소는 `https`가 아니라 `http`입니다. 브라우저 주소창에 `https://`로 입력하면 접속이 안 될 수 있으니 반드시 위 주소 그대로 접속합니다.

## 2. 다른 팀원이 지금 바로 테스트 가능한가?

가능합니다. 현재 기준으로 사용자 프론트, 관리자 프론트, API 서버가 모두 열려 있습니다.

다만 아래 경우에는 팀원 PC에서 접속이 안 될 수 있습니다.

| 상황 | 의미 | 조치 |
|---|---|---|
| 회사/개인 네트워크에서 특정 포트 차단 | `5173`, `5174`, `8000` 포트 접속이 막힌 상태 | 다른 네트워크에서 재시도하거나 인프라 담당자에게 포트 허용 확인 |
| AWS 보안 그룹 변경 | 서버 방화벽에서 외부 접속이 막힌 상태 | EC2 보안 그룹에서 `5173`, `5174`, `8000` 인바운드 허용 확인 |
| 스테이지 서버 재배포 중 | 배포 중 일시적으로 화면이 늦게 뜰 수 있음 | 1~3분 후 새로고침 |
| 테스트 계정 데이터가 반복 사용됨 | 수동 테스트를 여러 명이 하면 거래/문의/신고 이력이 누적됨 | 이는 정상이며, 필요 시 테스트 데이터 재주입 |

즉, 현재 배포 상태 기준으로 “당장 테스트가 불가능한 개발상 이유”는 없습니다. 접속이 안 된다면 대부분 네트워크, 포트, 보안 그룹, 또는 배포 중 상태 문제입니다.

## 4. 사용자 프론트 수동 검증 절차

### 4.1 기본 접속 및 로그인

1. 브라우저에서 http://43.201.76.192:5173 에 접속합니다.
2. 화면의 `LOGIN` 버튼을 클릭합니다.
3. `VIP Entrance` 로그인 창이 뜨는지 확인합니다.
4. `ops.smoke@longrise.ai` 원클릭 계정을 클릭합니다.
5. 화면에 `Operations Smoke` 또는 `ops_smoke`가 보이면 로그인 성공입니다.

성공 기준:

| 확인 항목 | 성공 기준 |
|---|---|
| 사용자 프론트 접속 | 메인 화면이 열린다 |
| 로그인 창 | `VIP Entrance`가 보인다 |
| 원클릭 로그인 | `Operations Smoke` 또는 `ops_smoke`가 보인다 |

### 4.2 패키지 및 구매

1. 로그인 후 상단 메뉴에서 `PACKAGES`를 클릭합니다.
2. `Dragon Wealth Packages` 화면이 보이는지 확인합니다.
3. `Flexible`, `VIP` 패키지가 보이는지 확인합니다.
4. `Flexible` 패키지를 클릭합니다.
5. `INVEST NOW`를 클릭합니다.
6. 약관 체크박스를 선택합니다.
7. `CONFIRM`을 클릭합니다.
8. `My Wealth` 또는 `PACKAGE HISTORY` 화면으로 이동하면 성공입니다.

성공 기준:

| 확인 항목 | 성공 기준 |
|---|---|
| 패키지 목록 | 실제 패키지명이 표시된다 |
| 구매 요청 | 확인 후 자산/이력 화면으로 이동한다 |
| 데이터 연결 | 화면이 멈추지 않고 API 결과가 반영된다 |
