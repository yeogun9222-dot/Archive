### 3️⃣ 제이크의 M13 EXIT 자동화 ✅ 설계 완료

**목표**: 13개월 만기 도달 회원의 정산과 알림을 완전 자동화

**산출물**:
- 🤖 자동화 플로우 4단계 (모니터링 → 정산 → 알림 → 로깅)
- 💻 JavaScript 구현 코드 템플릿 (크론 작업 포함)
- 🛡️ 안전 장치 3가지 (중복 방지, 오류 복구, 감사 로그)
- 📊 모니터링 대시보드 설계

**자동화 프로세스**:
```
Step 1: 매일 자정 UTC 기준 자동 모니터링
        → M13 도달 회원 감지

Step 2: 자동 정산 실행
        → 기본계산 + 15% 보장 적용
        → 최종환급액 결정

Step 3: 회원 자동 알림 (3채널)
        → SMS + 이메일 + 푸시 동시 발송
        → 인앱 알림 + 대시보드 업데이트

Step 4: 시스템 로그 기록
        → 감사 로그 저장
        → 관리자 대시보드 업데이트
```

**예시 시나리오** (VIP $5,000):
```
입금: 2026-01-01
M13: 2027-02-01 자동 정산 시작
  → 기본계산: -$15,463.87 (손실!)
  → 최소보장: $750 (15% 적용)
  → 최종환급: $750 ✅
  → 회사손실: -$16,213.87 (통제됨)

회원 즉시 알림:
  🎉 축하합니다! 프로그램 만기 도달
  💰 환급액: $750
  📅 정산일: 2027-02-01
```

**다음 단계**:
- [ ] 크론 작업 설정 (매일 00:00)
- [ ] 자동 정산 함수 구현
- [ ] 알림 시스템 통합

### 4️⃣ 실제 DB 연결 ✅ 설계 완료

**목표**: Mock 데이터에서 Production Database로 안전하게 전환

**산출물**:
- 🗄️ DB 스키마 설계 (5개 테이블 + 인덱스)
- 🔌 API 레이어 개발 계획 (TypeScript + axios)
- 🔄 마이그레이션 절차 (3단계)
- ✅ 검증 및 롤백 계획

**DB 구조**:
```
members (회원)
├─ id, user_id, balance, package
├─ join_date, status, rank
└─ created_at, updated_at

withdrawals (출금)
├─ id, member_id, amount, status
├─ request_date, wallet_address, network
└─ created_at

settlements (정산)
├─ id, member_id, settlement_type
├─ principal, basic_refund, final_refund
├─ floor_applied, months_elapsed
└─ created_at

daily_revenue (수익 추적)
├─ id, date (UNIQUE), total_revenue
└─ created_at

audit_logs (감사)
├─ id, action, user_id, member_id
├─ details (JSON)
└─ created_at
```

**마이그레이션 단계**:
```

                      ├─ API 엔드포인트 개발

                      └─ 에러 처리

Phase 3-3 (5/11~5/15): 데이터 마이그레이션
                       ├─ Mock 데이터 Export/Validate
                       ├─ DB 동기화 (트랜잭션)
                       └─ 무결성 검사

Phase 3-4 (5/16~5/20): 컴포넌트 통합
```
