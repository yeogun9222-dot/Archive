# 🚨 LONGRISE Critical Alerts (7가지)

**기준:** 정기점검(일주일 1회) 예정 시간 외에 발생하는 즉시 조치 필요 상황

---

## 1️⃣ 🔴 보안 침해 (Security Breach)

**상황:**
- 비정상적인 관리자 로그인 감지
- 대량 회원정보 다운로드 시도
- 지갑 주소 일괄 변경 감지
- API 토큰 탈취 시도

**왜 위험:**
- 회원 자산 탈취 가능
- 개인정보 유출
- 법적 책임

**대응시간:** ⚡ **즉시** (5분 이내)

**조치:**
```
1. 모든 관리자 세션 종료
2. 비밀번호 강제 초기화
3. 로그 보존 (증거)
4. 경찰청/사이버수사대 신고
5. 회원 공지
```

---

## 2️⃣ 🔴 시스템 다운 (System Outage)

**상황:**
- API 응답시간 > 5초
- 서버 응답 불가
- 회원들 거래 불가능
- 앱/웹 로그인 실패

**왜 위험:**
- 거래 중단 = 수익 정지 (시간당 수천 달러)
- 고객 이탈
- 신뢰도 하락

**대응시간:** ⚡ **5분 내**

**조치:**
```
1. DevOps 즉시 호출
2. 서버 상태 확인
3. 데이터베이스 연결 점검
4. 트래픽 분산
5. 백업 시스템 활성화
```

---

## 3️⃣ 🔴 뱅크런 (Liquidity Crisis)

**상황:**
- 일일 출금액 > 준비금의 70%
- 계산서: 내일 출금 못 해줄 수 있음

**왜 위험:**
- 회원들 자산 출금 불가
- 신뢰 붕괴
- 폰지사기 혐의
- 회사 파산

**대응시간:** ⏱️ **30분 내**

**조치:**
```
1. 출금 한도 임시 조정
2. 응급 자금 투입 검토
3. 경영진/재무팀 보고
4. 회원 공지 검토
5. 신규 입금 유도
```

---

## 4️⃣ 🔴 데이터 무결성 손상 (Data Corruption)

**상황:**
- 거래 기록 불일치
  - DB: 20건의 출금 신청
  - 실제 기록: 15건
- 로그와 현황이 다름

**왜 위험:**
- 누가 얼마를 출금해야 하는지 모호
- 중복 지급 또는 미지급
- 감사(Audit) 실패 → 운영 정지

**대응시간:** ⏱️ **1시간 내**

**조치:**
```
1. DB 전체 백업
2. 데이터 일관성 검사
3. 로그 vs 기록 대조
4. 불일치 항목 복구
5. 감사 기록 생성
```

---

## 5️⃣ 🟠 대규모 이상거래 (Anomaly: Suspicious Pattern)

**상황:**
- 자금세탁 의심 패턴
  - $100K 입금 → $99K 출금 → 반복
  - 1명이 24시간 500+ 거래
- 폰지사기 신호

**왜 위험:**
- AML(자금세탁방지) 법 위반
- 금융감시 기관 적발
- 운영 허가 취소
- 형사 처벌

**대응시간:** ⏱️ **2시간 내**

**조치:**
```
1. 해당 계정 즉시 동결
2. 거래 이력 보존 (증거)
3. FATF/FinCEN 신고
4. 법무팀 상담
5. 경찰청 신고
```

---

## 6️⃣ 🟠 규제 위반 적발 (Compliance Violation)

**상황:**
- 금융감시국(FCA/SEC) 적발
  - "얼굴인증 미완료 회원이 대액 거래"
  - 규정 위반
- 운영 정지 명령

**왜 위험:**
- 영업 정지
- 과태료 (수백만 달러)
- 경영진 개인 책임
- 회사 신뢰도 붕괴

**대응시간:** ⏱️ **2시간 내**

**조치:**
```
1. 해당 거래 즉시 회수
2. 미완료 회원 강제 얼굴인증
3. 규제 기관에 자진 신고
4. 법무팀 상담
5. 정책 개선
```

---

## 7️⃣ 🟡 대량 계정 잠금 (Mass Account Lock)

**상황:**
- 갑자기 100명+ 계정이 동시 잠김
- FDS 오류로 모두 거래 정지
- 회원들 자산 접근 불가

**왜 위험:**
- 회원 이탈 급증
- SNS에 "롱라이즈 사기" 폭주
- 법적 소송 위협
- 신뢰도 하락

**대응시간:** ⏱️ **3시간 내**

**조치:**
```
1. FDS 알고리즘 점검
2. 오류 여부 파악
3. 일괄 해제 또는 개별 복구
4. 회원 보상 검토
5. 언론 대응
```

## 🔧 **정기점검 정책 반영**

**정기점검 시간 외에 발생:**
- 위 7가지는 모두 Alerts 발송
- 자동 대응 시스템 작동

**정기점검 예정 시간에 발생:**
- 정기점검 일정에 병합 처리 가능한 것: 병합
- 긴급 대응 필요한 것(1-3순위): 정기점검 연기하고 즉시 대응
