## 📊 Admin Dashboard 메뉴 구조 (최종 확정)

### **1. Dashboard** (KPI 종합 대시보드)
- 실시간 KPI (총 회원, 활성 회원, 월 수익 등)

### **2. 회원 관리** (2가지 모두)
- **500명 회원 리스트 표시** (페이지네이션/가상 스크롤)
- **스마트 필터링** (복합 조건 검색)

### **3. 수익 관리** (2가지 모두)
- **일일 자동 지급** (스케줄 설정 및 자동 실행)
- **수동 지급** (개별/일괄 수동 처리)

### **4. 출금 관리** (3가지 모두)
- **승인** (미승인 출금 요청 검토 및 승인)
- **거부** (거부 이유 기록)
- **수동 처리** (직접 출금 처리)

### **5. 추천인 관리** (모두)
- 신청 승인/거부
- 추천인 코드 발급
- 추천인별 실적 추적

### **6. 팀 관리** (모두)
- 계보 트리 뷰
- 직급 수동 변경
- 팀 실적 분석

### **7. 보안 센터** (모두)
- 이상 징후 모니터링 및 알림
- 비정상 거래 탐지 (FDS)
- 계정 잠금/해제

### **8. 승인 센터** (모두)
- **얼굴인증 (Face Recognition) 승인** ⭐ (KYC 아님!)
- Distributor 신청 승인
- 기타 관리자 승인

### **9. 시스템 설정** (모두)
- 정책 변경 (최소출금, 수익률 등)
- 감시 규칙 설정
- 사용자 권한 관리

## 🔐 보안 정책 (필수)

### **1. 얼굴인증 (Face Recognition)**
- KYC 레벨 확인이 아님 → **얼굴 인증 (Liveness Check)**
- Admin의 승인센터에서 승인/거부 처리
- 승인 후 회원이 출금 가능

### **2. 거래비밀번호 (Trading Password)**
- **형식**: 영문 + 숫자 8자리 (필수)
- **설정 위치**: 회원의 프로필 페이지에서 설정
- **검증 순서**: 거래비밀번호 확인 → OTP 확인 → 출금 승인
- **필수 조건**: 거래비밀번호 설정 완료 후에만 출금 신청 가능

## 📋 기술 요구사항 (제이크 지시사항)

| 항목 | 스펙 |
|------|------|
| **최대 동시 로드** | 500명 |
| **필터링 응답 시간** | 1초 이내 |
| **백엔드 성능** | 위 스펙 기반 설계 |
| **일괄 작업 1000명 이상** | 2단계 승인 필수 |
| **2단계 승인** | 작업 요청 후 다른 Admin 재확인 |

---

## 🛠️ 3가지 필수 "치트키" 기능

1. **스마트 필터링 & 검색**: 조건 조합 + 1초 응답
2. **타임라인 로그**: 개별 회원의 모든 액션 기록
3. **일괄 작업 도구**: 수동 1000명 → 1클릭 자동화

## ✅ 메인 웹 사용자 여정 (Admin이 제어)

```
비회원 
  ↓ (이메일 + 6자리 인증코드)
회원가입
  ↓
로그인
  ↓
프로필 설정 → 거래비밀번호 설정 (필수) ⭐
  ↓
얼굴인증 신청 → Admin 승인 (필수) ⭐
  ↓
패키지 투자
  ↓
수익 확인 → Daily ROI 분배 (Admin 자동/수동) ⭐
  ↓
출금 신청
  ├─ 거래비밀번호 검증 ⭐
  ├─ OTP 검증
  └─ Admin 승인 ⭐
```
