## ⚠️ 팀 약점 (보완할 점)

### 1. 프로덕션 경험 ⚠️⚠️⚠️
**누가:** 전체 팀 (특히 신입~초급 3명)  
**약점:**
- 실제 상용 서비스 운영 경험 부족
- 대규모 동시 사용자 처리 미경험
- 실시간 모니터링/장애 대응 경험 미지수
- 성능 최적화 실전 경험 부족

**현재 수준:** 낮음~중간  
**보완 계획:**
```
Phase 1: 소규모 배포 (100명 규모)
→ Phase 2: 중규모 배포 (500명)
→ Phase 3: 전체 배포 (5000명+)

각 단계마다:
□ 장애 시뮬레이션
□ 성능 테스트
□ 모니터링 구축
□ 롤백 프로세스 연습
```

**담당:** 피오(Backend 리드), 제이크(PM 감시)  
**기한:** 3개월 내 경험 축적

---

### 2. 팀웍 & 협업 검증 ⚠️⚠️⚠️
**누가:** 전체 팀 (특히 신입 3명 + 기존 6명 조화)  
**약점:**
- 에바, 사라, 미나 3명 신규 추가 (팀웍 미검증)
- 크로스펑셔널 협업 프로세스 미정립
- 커뮤니케이션 채널 불명확
- 갈등 해결 매뉴얼 없음

**현재 수준:** 중간  
**보완 계획:**
```
주 1회: 팀 회의 (고정)
주 2회: UX팀 (에바+사라+미나) 내부 회의
월 1회: 팀 전체 회고 + 피드백

협업 규칙 정립:
□ 의사결정 프로세스 (언제, 누가, 어떻게?)
□ 피드백 문화 (비판적이되 존중하기)
□ 문제 발생 시 에스컬레이션 (제이크 → 사용자)
```

**담당:** 제이크(PM)  
**기한:** 2주 내 정립, 지속 개선

### 3. 도메인 지식 (LONGRISE) ⚠️⚠️
**누가:** 전체 팀  
**약점:**
- LONGRISE 마스터플랜 V7.4 깊이 부족
- 금융 상품 이해도 낮음
- 사용자 심리 이해 미숙
- 경쟁사 분석 부족

**현재 수준:** 낮음  
**보완 계획:**
```
오리엔테이션 (1주일):
□ V7.4 마스터플랜 교육 (제이크)
□ 금융 상품 이해 (카이)
□ 사용자 인터뷰 (에바+사라)
□ 경쟁사 분석 (카이)

지속 학습:
□ 주 1회 사용자 피드백 공유
□ 월 1회 수익/손실 분석
□ 분기별 마스터플랜 복습
```

**담당:** 제이크(PM 주도), 카이(GTM 협력)  
**기한:** 3개월 내 숙달
